## 문제 해결 과정 알고리즘

```
1. 문제를 읽고 이해한다.
2. 문제를 익숙한 용어로 재정의 한다.
3. 어떻게 해결할지 계획을 세운다.
4. 계획을 검증한다.
5. 프로그램으로 구현한다.
6. 복기하고, 개선 방법을 찾아본다.
```

1. 문제를 읽고 이해하기
  
    입출력 예제 등만으로 판단하지 말고 문제를 완전히 이해할 것

2. 문제를 익숙한 용어로 재정의 한다.
   
   현실 세계의 개념을 다루기 용이한 수학적/전산학적 개념으로 옮겨 표현한다.

3. 계획 세우기
   
   어떤 방식으로 문제를 해결할지 결정하고, 알고리즘/자료구조를 선택한다.
4. 계획 검증
   
   설계한 알고리즘이 요구 조건을 정확히 수행하는지 증명하고, 시간/메모리 제한을 준수하는지 확인한다.

5. 구현
   
6. 회고
   
   자신이 문제를 해결한 과정을 돌이켜 보고 개선하는 과정.
   
   문제를 풀 때 코드와 함께 경험을 기록으로 남길 것. 간단한 해법과, 접근 방식, 해법을 찾는데 결정적이었던 깨달음은 무엇인지 등.

   오답 원인 과 다른 사람의 코드를 보는 것도 효과적

   9. 순서를 강제할 수 있는가?
