## 8.3. `nn.Module` 서브클래싱하기

기존 모듈의 이어붙이기만으로는 한계가 옴. skip-connection이나, 다른 연산들이 필요한 경우가 생김

가장 일반적인 구조는 아래와 같음.

```python
class Net(nn.Module):
	def __init__(self, ...):
		# todo : 필요한 레이어나, 연산 등
		# EX
		# self.conv1 = nn.Conv2d(3, 16, kernel_size = 3, padding = 1)
		# self.activation1 = nn.Tanh()
		# ...
	def forward(self, x):
		# todo : 레이어 어떻게 통과할건지?
```

참고) 위의 커스텀 네트워크 안의 모듈은 서브 모듈이 되는데, 이는 리스트나, 딕셔너리 내에 있으면 옵티마지어가 서브모듈과 파라미터를 찾지 못한다. 필요하다면 `nn.MoudleList` 또는 `nn.ModuleDict` 를 사용해야 한다.

### 8.3.3. 함수형 API

`self.activation1 = nn.Tanh()` 애들 처럼 파라미터가 필요 없는 서브모듈은 굳이 내가 정의할 필요가 있나? 직접호출이 낫지 않나?? → 그래서 준비했습니다! 함수형 API!(함수형? 내부 상태가 없음 = 출력이 전적으로 입력에 의해 결정 됨)

`nn.functional` 에서는 `nn` 에서 찾을 수 있는 많은 모듈의 함수형을 제공. 단 입력에 의해 결정되기 때문에 파라미터도 받아야함 (ex. `nn.F.linear(input, weight, bias = None)` ; 참고 ; 대게 as F를 통해 호출함.)

단, 모듈 API를 사용하는 것이 좋을 때도 있는데, 여러 애플리케이션에서 필요한 경우가 그렇다. 같은 모듈 인스턴스를 재사용하면, 파이토치 용법에서는 문제가 없으나, 모델을 분석하는 도구가 문제를 일으킬 수 있다. 이렇게 되면 양자화나, 모델을 압축하거나 할 때 문제가 발생한다.

## 8.5. 모델 설계

### 8.5.2. ! 모델이 수렴하고 일반화하도록 돕는 방법 : 정규화(Regularization)

- 파라미터 제어: 가중치 패널티
    - L2정규화 코드 : → 할필요 없음. SGD클래스에서 weight_decay파라미터를 설정하면 됨!
    
    ```python
    # 훈련 루프에서...
    l2_lambda = 0.001 # 페널티 정도
    l2_norm = sum(p.pow(2.0).sum()
    							for p in model.parameter())
    # L1정규화의 경우 위의 줄에서 pow(2.0)을 abs()로 변경
    loss = loss + l2_lambda * l2_norm
    
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    ```
    
- Dropout : `nn.Dropoutnd(p=probability)`
- Batch Normalization : `nn.BatchNormnd(num_features)`
초기화에 영향을 덜 받고, 미니 배치마다 정규화를 하기 때문에 일종의 데이터 증강과 같은 효과를 볼 수 있다.

### 9.4.1. 신경망이 동작할 때까지 데이터를 던져넣을 수 없는 이유

왜 동시에 두 개의 분리된 모델 아키텍쳐가 존재하고, 데이터 흐름은 복잡하고... → 자동화하기 어려운 문제이기 때문에, 사람들이 제대로 파악하지 못함.

그럼 왜 어려운감..?

'악성 종양이 있는가?' 라는 질문에 대해 대부분의 CT스캔은 근본적으로 흥미롭지 않은 데이터. 대부분의 환자의 몸은 건강한 세포로 이뤄져 있기 때문. 악성 종양이 있는 경우에서 99.9999%는 정상 세포이다. 물론 입력을 넣으면 통으로 결절 위치를 파악하고, 결절 여부를 판단하는 모델을 만들수도 있겠으나.. 연산 자원도 고려해야하며, 막대한 데이터 역시 필요함. 최선의 방법은 데이터를 더 잘 통합할 수 있는 검증된 모델 설계를 검토하는 것.
